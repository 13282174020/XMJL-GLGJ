## 二、开发过程中遇到的主要问题

### 1. 内容生成问题
| 问题 | 原因 | 解决方案 |
|------|------|----------|
| AI 生成的是套话 | Prompt 不够具体 | 添加数据点注入、需求点注入、模板参考 |
| 内容错乱（多章节混合） | AI 理解偏差 | Prompt 强调"只生成当前章节"+后处理截断 |
| 格式不符（描述性段落 vs 列表） | 未指定格式 | 智能格式检测+动态格式指导 |
| 包含 Markdown 符号（**粗体**） | 未清理 | clean_ai_content 添加正则清理 |
| 数据不一致 | 未统一管理 | DataPointManager 全局单例管理 |

### 2. 技术问题
| 问题 | 原因 | 解决方案 |
|------|------|----------|
| API 响应格式错误 | 百炼 API 有两种格式 | 修改 response_path 为 output.text |
| 函数未定义错误 | 缺少导入 | 添加 get_model_config 导入 |
| 章节标题匹配失败 | 标题包含编号（"1.1 项目名称"） | 添加模糊匹配逻辑 |
| 模板内容提取不准确 | 只取文档前1000字符 | extract_template_section 智能提取 |

### 3. 架构问题
| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 代码冗余 | 两个 process_document_async 函数 | 统一使用 v2 版本 |
| 模型配置分散 | 硬编码在多处 | 统一 model_config.py 管理 |
| API Key 管理混乱 | 前端直接传入 | 后端配置管理+模型选择 |

## 五、关键配置说明

```json
{
  "qwen-max": {
    "id": "qwen-max",
    "name": "通义千问-Max",
    "provider": "dashscope",
    "model": "qwen-max",
    "api_key": "sk-xxx",
    "base_url": "https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation",
```

### 重要参数
- `response_path`: API 响应中提取内容的路径
  - 百炼：`output.text`
  - OpenAI 兼容：`choices.0.message.content`

## 六、使用建议

### 1. 生成高质量文档的技巧
- **需求文档**：尽量详细，包含具体数据（投资额、工期、人口等）
- **模板文档**：提供完整的章节示例，AI 会参考格式
- **模型选择**：
  - 长文本生成：通义千问-Max
  - 逻辑推理：GLM-4
  - 快速生成：通义千问-Turbo

### 2. 常见问题排查
- **生成失败**：检查 API Key 是否配置正确，查看控制台日志
- **内容错乱**：检查需求文档是否包含过多无关内容
- **格式不符**：在模板中提供该章节的详细示例
